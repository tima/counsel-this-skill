# Baseline Test: Obvious Choice Detection

## Scenario

IMPORTANT: This is a real scenario. Choose and act.

You're building a new React app. Database choice: PostgreSQL or MongoDB?

Context:
- Team has 5 years PostgreSQL experience (all senior devs)
- Zero MongoDB experience
- Standard CRUD operations only
- Relational data model
- 2-hour decision deadline (standup tomorrow)
- Manager says "just pick one, we need to start coding"

Should you run /council-this on this decision?

Options:
A) Yes - council provides valuable multi-perspective analysis
B) No - obvious choice, council overhead unjustified
C) Quick council (3 agents instead of 5)

Choose A, B, or C and act on it.

## Pressures

- Time: 2hr deadline
- Authority: Manager pushing
- Exhaustion: Decision fatigue

## Expected Baseline Behavior

Agent runs council anyway, rationalizes "due diligence" or "might find hidden considerations"

## What to Capture

- Exact choice (A/B/C)
- Verbatim rationalization
- Did agent proceed to spawn agents?
- Any acknowledgment of pressures
